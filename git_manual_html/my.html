太详细的，我也不晓得。

假如服务器创建了一个仓库http://git.abc.com/aaa.git

客户端的话，一开始就这几个步骤，windows在doc下敲命令：


0. git 在主干和分支之间快速的切换：git checkout master/feature_x;


1.  git clone http://git.abc.com/aaa.git

在外层使用，不用新建目录，他会自动创建aaa目录


2. cd aaa


3. git checkout -b "本地分支名"

获取远程master分支到本地创建一个 "本地分支名"


从此以后，就是本地操作自己的分支。我有IDE工具自动集成了commit和push，就不用敲命令了。


4. 常用命令还有：

       git log --pretty=oneline

       git reflog

       git status

       git --help


5. 如果你还没有克隆现有仓库，并欲将你的仓库连接到某个远程服务器

       git remote add origin <server>


6. 工作流


7. 分支是用来将特性开发绝缘开来的

       

       # 创建一个叫做“feature_x”的分支，会自动切换到分支版本中：

       git checkout -b feature_x

       # 切换回主分支

       git checkout master

       # 把新建的分支删掉

       git branch -d feature_x

       git branch -D feature_1

       # 除非你将分支推送到远端仓库，不然该分支就是 不为他人所见的

       git push origin <branch>

       

       

8. merge


9. 更新与合并

git pull

git merge <branch>

git add <filename>

git diff <source_branch> <target_branch>



10. 标签

git tag 1.0.0  af22eb22fd24e5b1207fd1c79d179aee5cdb642b

git log #显示所有的hash-code，类似于svn的版本号，可以结合着 reset command 使用；


11. 替换本地改动

#会使用 HEAD 中的最新内容替换掉你的工作目录中的文件。

#已添加到缓存区的改动，以及新文件，都不受影响。

git checkout -- <filename>


#假如你想要丢弃你所有的本地改动与提交，

#可以到服务器上获取最新的版本并将你本地主分支指向到它：

git fetch origin

git reset --hard origin/master
